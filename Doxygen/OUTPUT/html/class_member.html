Detailed Description

This is the Member base class. It represents all the common data all members of a gym have. Number of basic functions are included allowing setting and retrieving data for a member type. Currently, there are two types of Members: CUSTOMER and STAFF. This is represented using enum Type.

Any subclass of member must implement a print, clone, and serialize function. The print() function simply prints out any data associated with that specific member to the console. The clone() function returns a copy of itself, used by the prototype pattern The serialize() function turns the data associated with a member into a structured form to be stored in a file.
